# MaxToTelega

Мост между мессенджером **MAX** и **Telegram**: входящие сообщения твоего
личного аккаунта MAX пересылаются тебе в Telegram-бота, а ответы из Telegram
(реплаем) уходят обратно в нужный чат MAX. Поддерживаются текст, фото, файлы и
видео в обе стороны.

```
   MAX (личный аккаунт)            Telegram-бот (ты)
        │  входящее                     │
        ▼                               ▼
   ┌─────────────────────────────────────────┐
   │              bridge.py                   │
   │  PyMax userbot  ⇄  aiogram bot           │
   │  + SQLite-маршрутизация ответов          │
   └─────────────────────────────────────────┘
        ▲                               ▲
        │  отправка ответа              │  reply на сообщение
```

## ⚠️ Важно прочитать перед использованием

Официальный Bot API мессенджера MAX **не даёт доступа к личной переписке** —
он работает только с сообщениями, адресованными боту. Чтобы зеркалить личный
аккаунт, мост использует **неофициальную** библиотеку
[PyMax](https://github.com/MaxApiTeam/PyMax), которая логинится под твоим
аккаунтом по внутреннему протоколу MAX.

Из этого следуют риски, которые ты принимаешь на себя:

- Это **неофициальный протокол**. Он не документирован и может **сломаться** при
  любом обновлении MAX — придётся обновлять PyMax / чинить мост.
- Использование userbot **противоречит правилам MAX**; теоретически возможна
  блокировка аккаунта. Веди себя «по-человечески» — без массовых рассылок и
  флуда.
- Файл сессии MAX (`data/`) и `.env` — это **полный доступ к твоему аккаунту**.
  Не коммить их и храни на сервере под ограниченными правами.

Используй мост только со **своим** аккаунтом.

## Требования

- Python **3.10+** (PyMax требует `>=3.10`)
- Telegram-бот (токен от [@BotFather](https://t.me/BotFather))
- Аккаунт MAX с доступом по номеру телефона

## Установка

```bash
git clone https://github.com/kostya4118/MaxToTelega.git
cd MaxToTelega

python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# отредактируй .env (см. ниже)
```

## Настройка `.env`

| Переменная           | Что это                                                         |
| -------------------- | -------------------------------------------------------------- |
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather                                        |
| `TELEGRAM_OWNER_ID`  | Твой Telegram user id (числом). Только с него принимаются ответы |
| `MAX_PHONE`          | Номер телефона аккаунта MAX, формат `+79990000000`             |
| `FORWARD_GROUPS`     | `true` — пересылать и группы/каналы; `false` — только диалоги    |
| `MARK_READ`          | `true` — отмечать сообщение в MAX прочитанным, когда отвечаешь    |
| `MAX_PROXY`          | Прокси для MAX, если сервер не в стране телефона (см. ниже)        |
| `TELEGRAM_GROUP_ID`  | id группы-форума для режима тем (опционально, см. ниже)           |

### Режим тем (каждый чат MAX — отдельная тема)

По умолчанию все сообщения приходят в личку с ботом. Если хочется, чтобы каждый
чат MAX был отдельным «подчатом» (как темы в Telegram-форуме):

1. Создай группу и включи в её настройках **Темы** (Topics).
2. Добавь туда бота и сделай его **админом** с правом **«Управление темами»**.
3. Напиши в группе `/id` — бот покажет «ID этой группы». Впиши его в
   `TELEGRAM_GROUP_ID` и перезапусти мост.

Дальше мост сам заводит по теме на каждый чат MAX (диалог → тема с именем
человека, группа → тема с названием группы). Чтобы ответить — просто пиши
**в нужной теме** (reply не нужен). `/mute` внутри темы заглушает её чат.

### Прокси (если сервер в другой стране)

MAX блокирует вход как «подозрительный», если он приходит с IP в другой стране,
чем твой телефон (вплоть до завершения всех сессий и отказа слать SMS). Решение —
указать `MAX_PROXY` с IP в твоём регионе. Рекомендуется **HTTP(S)-прокси** — он
работает и для входа, и для выгрузки вложений:

```
MAX_PROXY=http://user:pass@1.2.3.4:8080
```

SOCKS5 (`socks5://…`) поднимет переписку, но выгрузка фото/файлов в MAX может не
работать (её PyMax шлёт через aiohttp, который SOCKS не понимает).

Не знаешь свой `TELEGRAM_OWNER_ID`? Оставь пустым, запусти мост, напиши боту
команду `/id` — он пришлёт твой id. Впиши его в `.env` и перезапусти.

## Первый запуск

```bash
python bridge.py
```

При первом запуске MAX попросит **SMS-код** прямо в консоли (и пароль 2FA, если
он включён). После успешного входа сессия сохраняется в `data/`, и дальше код
больше не спрашивается. Поэтому первый запуск делай **интерактивно в терминале**,
а не сразу как фоновый сервис.

## Как пользоваться

- Любое входящее в MAX приходит в Telegram с заголовком «💬 _кто/какой чат_».
- Чтобы ответить — сделай **reply** на это сообщение в Telegram. Текст и/или
  вложение уйдут в тот же чат MAX.
- Просто написать боту (без reply) нельзя — мост не знает, в какой чат слать.

### Уведомления по чатам

По умолчанию всё приходит со звуком. Управление — командами в Telegram:

| Команда   | Как применить                                  | Что делает                          |
| --------- | ---------------------------------------------- | ----------------------------------- |
| `/mute`   | **reply** на пересланное сообщение из чата      | этот чат MAX → доставка без звука    |
| `/unmute` | **reply** на пересланное сообщение из чата      | вернуть чат со звуком                |
| `/muted`  | просто отправить боту                          | показать список заглушённых чатов    |

Настройка хранится в базе и переживает перезапуск.

## Запуск в Docker

Самый простой способ для VPS. Первый запуск — **интерактивно**, чтобы ввести
SMS-код:

```bash
cp .env.example .env   # заполни TELEGRAM_BOT_TOKEN и MAX_PHONE
docker compose build
docker compose run --rm bridge   # введёшь SMS-код, дождись «client started», Ctrl+C
```

После этого сессия лежит в `./data`, и дальше можно гонять в фоне:

```bash
docker compose up -d
docker compose logs -f
```

## Запуск как сервис (systemd, Linux VPS)

После того как первый интерактивный запуск создал сессию в `data/`:

```ini
# /etc/systemd/system/maxtotelega.service
[Unit]
Description=MaxToTelega bridge
After=network-online.target
Wants=network-online.target

[Service]
WorkingDirectory=/opt/MaxToTelega
ExecStart=/opt/MaxToTelega/.venv/bin/python bridge.py
Restart=always
RestartSec=5
User=maxbridge

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now maxtotelega
journalctl -u maxtotelega -f
```

## Структура

| Файл               | Назначение                                          |
| ------------------ | --------------------------------------------------- |
| `bridge.py`        | Точка входа: связывает MAX-userbot и Telegram-бота   |
| `config.py`        | Загрузка конфигурации из `.env`                     |
| `storage.py`       | SQLite-маршрутизация ответов Telegram → MAX         |
| `requirements.txt` | Зависимости                                         |

## Что умеет

- Текст, фото, файлы, видео, **стикеры** и **аудио** из MAX → Telegram.
- Несколько фото/видео в одном сообщении приходят **альбомом** (media group).
- Текст, фото, документы и видео из Telegram → обратно в MAX (реплаем).
- **Отметка «прочитано»** в MAX, когда ты отвечаешь (отключается `MARK_READ`).
- Индикатор активности в Telegram, пока мост качает вложения.
- **Тихие чаты:** заглушай отдельные чаты MAX командой `/mute` (см. ниже).

## Ограничения и идеи на будущее

- Контакты, опросы, видеосообщения-«кружки» и т.п. из MAX пока приходят как
  пометка/файл, не разворачиваются полностью.
- Анимированные стикеры MAX (Lottie) отправляются как статичная картинка.
- Альбом, отправленный из Telegram в MAX, уйдёт как несколько отдельных
  сообщений (входящие альбомы MAX→TG группируются корректно).
- Видео и файлы ограничены лимитами Telegram Bot API (~50 МБ на отправку ботом).
- «Печатает» на стороне MAX не поддержано: PyMax не выводит этот опкод
  высокоуровнево, а угадывать недокументированный формат рискованно.
- Идеи: пересылка реакций, индикатор «печатает» в MAX, альбомы TG→MAX.
