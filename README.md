# MaxToTelega

Мультитенантный мост между мессенджером **MAX** и **Telegram**. Один Telegram-бот
обслуживает многих пользователей: каждый добавляет свои MAX-аккаунты **прямо через
бота** (вход по SMS/2FA вводится в чат). У каждого MAX-аккаунта — своя
группа-форум, где каждый чат MAX становится отдельной темой. Ответы из темы
уходят обратно в нужный чат MAX. Поддерживаются текст, фото, файлы, видео,
стикеры и аудио в обе стороны.

```
  пользователь A ─┐                              ┌─ группа A (форум, темы)
  пользователь B ─┤   один Telegram-бот (bridge) ├─ группа B (форум, темы)
       …          ┘   ├ userbot MAX на аккаунт    └─ …
                      └ registry.db + базы по аккаунтам
```

## ⚠️ Важно прочитать перед использованием

Официальный Bot API мессенджера MAX **не даёт доступа к личной переписке**. Чтобы
зеркалить аккаунт, мост использует **неофициальную** библиотеку
[PyMax](https://github.com/MaxApiTeam/PyMax), которая логинится под аккаунтом по
внутреннему протоколу MAX. Отсюда риски:

- Это **неофициальный протокол** — может **сломаться** при обновлении MAX.
- Userbot **противоречит правилам MAX**; возможна блокировка аккаунта. Без
  массовых рассылок/флуда риск ниже, но не нулевой.
- **Ты держишь чужие сессии.** Хранишь на сервере сессии MAX всех пользователей —
  это **полный доступ к их аккаунтам**. Утечка `data/` = компрометация всех.
  Береги сервер, ограничь права, по возможности шифруй диск.
- **Регистрация модерируется.** Новый пользователь не войдёт, пока админ
  (`ADMIN_TG_ID`, по умолчанию `TELEGRAM_OWNER_ID`) не одобрит заявку кнопкой в
  чате. Сам админ добавляет аккаунты без подтверждения.
- Каждый аккаунт входит **с IP сервера**. Если сервер не в стране пользователя,
  MAX может счесть вход подозрительным (см. «Прокси»).

## Требования

- Python **3.10+**
- Telegram-бот (токен от [@BotFather](https://t.me/BotFather))

## Установка и настройка с нуля

Пошагово — для оператора, который поднимает бота на своём Linux-сервере.

### Шаг 1. Создай Telegram-бота

1. Открой [@BotFather](https://t.me/BotFather) → `/newbot` → задай имя и
   username (`..._bot`). BotFather пришлёт **токен** вида `123456:ABC-DEF…` —
   сохрани его.
2. Там же настрой бота (по желанию, но удобно):
   - `/setprivacy` → выбери бота → **Disable** (чтобы видел команды в группах).
   - `/setcommands` → выбери бота → вставь:
     ```
     add - добавить MAX-аккаунт
     accounts - мои аккаунты
     remove - удалить аккаунт
     setproxy - задать прокси аккаунту
     relogin - повторить вход
     bind - привязать группу (в группе)
     mute - заглушить тему
     unmute - вернуть звук
     muted - список заглушённых
     help - помощь
     ```

### Шаг 2. Узнай свой Telegram id (ты будешь админом)

Напиши [@userinfobot](https://t.me/userinfobot) — он пришлёт твой числовой id.
Он нужен для `ADMIN_TG_ID`: только админ одобряет чужие регистрации (и сам
добавляет аккаунты без подтверждения).

### Шаг 3. Подготовь сервер

Команды для Debian/Ubuntu. Проверь Python (нужен **3.10+**): `python3 --version`.

```bash
# отдельный системный пользователь без логина — мост живёт под ним
sudo useradd --system --create-home --home-dir /opt/maxtotelega \
     --shell /usr/sbin/nologin maxbridge

# код и виртуальное окружение (НИКОГДА не ставь зависимости в системный python)
sudo -u maxbridge git clone https://github.com/kostya4118/MaxToTelega.git \
     /opt/maxtotelega/app
cd /opt/maxtotelega/app
sudo -u maxbridge python3 -m venv .venv
sudo -u maxbridge ./.venv/bin/pip install --upgrade pip
sudo -u maxbridge ./.venv/bin/pip install -r requirements.txt
```

### Шаг 4. Заполни `.env`

```bash
sudo -u maxbridge cp .env.example .env
sudo -u maxbridge nano .env
```

Минимум, что нужно вписать:

```ini
TELEGRAM_BOT_TOKEN=123456:ABC-DEF...   # из шага 1
ADMIN_TG_ID=123456789                  # твой id из шага 2
```

Закрой файл от посторонних (в нём токен):

```bash
sudo chmod 600 /opt/maxtotelega/app/.env
sudo chown maxbridge:maxbridge /opt/maxtotelega/app/.env
```

### Шаг 5. Запусти как сервис (systemd)

Вход в MAX теперь идёт через Telegram, поэтому интерактивный запуск в консоли
не нужен — сразу делаем сервис.

```bash
sudo tee /etc/systemd/system/maxtotelega.service > /dev/null <<'EOF'
[Unit]
Description=MaxToTelega bridge
After=network-online.target
Wants=network-online.target

[Service]
User=maxbridge
Group=maxbridge
WorkingDirectory=/opt/maxtotelega/app
ExecStart=/opt/maxtotelega/app/.venv/bin/python -u bridge.py
Restart=always
RestartSec=10
MemoryMax=400M
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/opt/maxtotelega/app/data

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now maxtotelega
sudo journalctl -u maxtotelega -f
```

В логах хочешь увидеть `Запуск мультитенантного моста…` и
`Run polling for bot @твойбот`. Бот готов принимать `/add`.

### Шаг 6. Проверь, что бот живой

Напиши боту в личке `/start` — он ответит списком команд. Дальше — добавляй
аккаунты (раздел «Как пользоваться» ниже).

### Обновление в будущем

```bash
cd /opt/maxtotelega/app
sudo -u maxbridge git pull
sudo systemctl restart maxtotelega
```
Зависимости меняются редко; если в коммите тронут `requirements.txt` —
`sudo -u maxbridge ./.venv/bin/pip install -r requirements.txt` перед рестартом.

## Как пользоваться (для каждого пользователя)

1. **Добавить аккаунт.** Напиши боту в личке `/add` → пришли номер телефона MAX
   (`+79991234567`) → бот попросит **код из SMS** (пришли его в чат) → если есть
   2FA, попросит **пароль приложения**.
2. **Создать группу.** После успешного входа создай в Telegram **группу**, включи
   в её настройках **Темы (Topics)**, добавь туда бота и сделай его **админом**
   с правом **«Управление темами»**.
3. **Привязать группу.** В этой группе напиши `/bind` — бот привяжет её к твоему
   аккаунту. Если аккаунтов несколько — `/bind N` (номер из `/accounts`).
4. Готово: входящие из MAX появляются темами в группе. Чтобы ответить — пиши
   **в нужной теме** (reply не нужен), бот ставит 👍 при успешной отправке.

Команды:

| Команда | Где | Что делает |
|---|---|---|
| `/add` | личка | добавить MAX-аккаунт (телефон + SMS + 2FA) |
| `/accounts` | личка | список твоих аккаунтов и статус |
| `/remove N` | личка | удалить аккаунт N (сессию и темы) |
| `/setproxy N <url>` | личка | задать прокси аккаунту (`off` — убрать) |
| `/relogin N` | личка | повторить вход (после прокси / сброса сессии) |
| `/bind [N]` | группа | привязать группу к аккаунту |
| `/mute` / `/unmute` | тема | заглушить/вернуть звук этого чата |
| `/muted` | группа | список заглушённых чатов |

Несколько аккаунтов у одного пользователя — просто повтори `/add` и привяжи
каждому свою группу.

## Админ-команды (только для `ADMIN_TG_ID`)

| Команда | Что делает |
|---|---|
| `/admin` | дашборд: пользователей, аккаунтов, статусы, баны |
| `/admin list` | все аккаунты (id, владелец, телефон, группа, статус) |
| `/admin user <tg_id>` | аккаунты конкретного пользователя |
| `/admin stop N` / `start N` / `remove N` | остановить / перезапустить / удалить любой аккаунт |
| `/ban <tg_id>` | забанить пользователя (его аккаунты останавливаются, новые `/add` запрещены) |
| `/unban <tg_id>` | снять бан |
| `/banned` | список забаненных |
| `/backup` | собрать бэкап и прислать архивом в Telegram |

### Бэкап

Самое ценное и невосстановимое — **сессии MAX** (без них аккаунты пришлось бы
заводить заново через SMS), плюс реестр и привязки тем. Всё это в `data/*.db`.

- **`/backup`** (админ) — собирает консистентный архив всех БД и присылает его
  тебе в Telegram (готовая офсайт-копия). Если архив больше ~50 МБ — сохраняет на
  сервере и пишет путь.
- **Авто-бэкап** по таймеру `BACKUP_INTERVAL_HOURS` (по умолчанию раз в сутки) в
  `data/backups/`, хранятся последние `BACKUP_KEEP` (по умолчанию 7).
- **Восстановление:** распакуй архив и положи `*.db` обратно в `WORK_DIR`
  (`tar -xzf backup_*.tar.gz` → файлы из `data/` в свой `data/`), затем
  перезапусти сервис. `.env` бэкапится отдельно (там только токен/настройки).

Регистрации чужих пользователей приходят админу запросом с кнопками
**Одобрить/Отклонить** — вход в MAX стартует только после одобрения. Антифлуд
(`ADD_COOLDOWN_SEC`, `ADD_MAX_PER_HOUR`, `MAX_ACCOUNTS_PER_USER`) ограничивает
спам регистраций. Сам админ от лимитов и одобрения освобождён.

## Прокси (если сервер в другой стране, чем телефон)

MAX блокирует вход с «чужого» IP (вплоть до отказа слать SMS). Тогда аккаунту
нужен **HTTP(S)-прокси** с IP в нужном регионе. Задаётся прямо из бота:

```
/setproxy N http://user:pass@1.2.3.4:8080   # задать
/setproxy N off                              # убрать
/relogin N                                   # повторить вход через прокси
```

Если при `/add` вход не удался (нет SMS из-за гео), аккаунт **не удаляется** —
бот подскажет задать прокси и повторить `/relogin N`. SOCKS5 поднимет переписку,
но выгрузка файлов в MAX через него может не работать — лучше HTTP(S).

## Переезд со старой single-версии

Если раньше был один аккаунт из `.env` — заполни в `.env` блок миграции
(`TELEGRAM_OWNER_ID`, `MAX_PHONE`, `TELEGRAM_GROUP_ID`, при необходимости
`MAX_SESSION`, `MAPPING_DB`, `MAX_PROXY`). При первом запуске с **пустым реестром**
этот аккаунт перенесётся автоматически, с сохранением сессии и тем — повторно
входить не придётся.

## Структура

| Файл | Назначение |
|---|---|
| `bridge.py` | Менеджер: онбординг аккаунтов, маршрутизация, жизненный цикл |
| `registry.py` | Реестр аккаунтов (кто/телефон/группа/сессия) |
| `storage.py` | Per-account база: темы и маршрутизация ответов |
| `config.py` | Общие настройки из `.env` |

## Что умеет

- Текст, фото, файлы, видео, **стикеры** и **аудио** из MAX → Telegram.
- Несколько фото/видео — **альбомом**; нерисуемые вложения — текстовой пометкой.
- Ответы из тем → обратно в MAX (текст, фото, документы, видео).
- Темы автоматически именуются по чату/собеседнику и переименовываются, когда
  появляется имя; чаты-боты помечаются `🤖 бот`.
- Тихие чаты (`/mute`), авто-ротация логов, чистое завершение, авто-перезапуск
  аккаунтов после рестарта.

## Ограничения

- Отметка «прочитано» отключена (текущий MAX рвёт на неё соединение).
- Контакты, опросы, «кружки» из MAX приходят пометкой, не разворачиваются.
- Анимированные стикеры (Lottie) — статичной картинкой.
- Видео/файлы ограничены лимитами Telegram Bot API (~50 МБ на отправку ботом).
- Прокси для аккаунтов, добавленных через бота, пока задаётся только через
  реестр/миграцию (не из диалога).
