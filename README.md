# MaxToTelega

Мультитенантный мост между мессенджером **MAX** и **Telegram**. Один Telegram-бот
обслуживает многих пользователей: каждый добавляет свои MAX-аккаунты **прямо через
бота** (вход по SMS/2FA вводится в чат). У каждого MAX-аккаунта — своя
группа-форум, где каждый чат MAX становится отдельной темой. Ответы из темы
уходят обратно в нужный чат MAX. Поддерживаются текст, фото, файлы, видео,
стикеры и аудио в обе стороны.

```
  пользователь A ─┐                              ┌─ группа A (форум, темы)
  пользователь B ─┤   один Telegram-бот (bridge) ├─ группа B (форум, темы)
       …          ┘   ├ userbot MAX на аккаунт    └─ …
                      └ registry.db + базы по аккаунтам
```

## ⚠️ Важно прочитать перед использованием

Официальный Bot API мессенджера MAX **не даёт доступа к личной переписке**. Чтобы
зеркалить аккаунт, мост использует **неофициальную** библиотеку
[PyMax](https://github.com/MaxApiTeam/PyMax), которая логинится под аккаунтом по
внутреннему протоколу MAX. Отсюда риски:

- Это **неофициальный протокол** — может **сломаться** при обновлении MAX.
- Userbot **противоречит правилам MAX**; возможна блокировка аккаунта. Без
  массовых рассылок/флуда риск ниже, но не нулевой.
- **Ты держишь чужие сессии.** Запустив открытый бот, ты на своём сервере
  хранишь сессии MAX всех пользователей — это **полный доступ к их аккаунтам**.
  Утечка папки `data/` = компрометация всех. Береги сервер, ограничь права,
  по возможности шифруй диск. Это серьёзная ответственность.
- Каждый аккаунт входит **с IP сервера**. Если сервер не в стране пользователя,
  MAX может счесть вход подозрительным (см. «Прокси»).

## Требования

- Python **3.10+**
- Telegram-бот (токен от [@BotFather](https://t.me/BotFather))

## Установка

```bash
git clone https://github.com/kostya4118/MaxToTelega.git
cd MaxToTelega
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env       # впиши TELEGRAM_BOT_TOKEN
python bridge.py
```

Для бота у @BotFather желательно **включить inline/группы** и затем сделать бота
админом нужных групп (см. ниже). Команды можно добавить через BotFather для
удобства: `add, accounts, remove, bind, mute, unmute, muted`.

## Как пользоваться (для каждого пользователя)

1. **Добавить аккаунт.** Напиши боту в личке `/add` → пришли номер телефона MAX
   (`+79991234567`) → бот попросит **код из SMS** (пришли его в чат) → если есть
   2FA, попросит **пароль приложения**.
2. **Создать группу.** После успешного входа создай в Telegram **группу**, включи
   в её настройках **Темы (Topics)**, добавь туда бота и сделай его **админом**
   с правом **«Управление темами»**.
3. **Привязать группу.** В этой группе напиши `/bind` — бот привяжет её к твоему
   аккаунту. Если аккаунтов несколько — `/bind N` (номер из `/accounts`).
4. Готово: входящие из MAX появляются темами в группе. Чтобы ответить — пиши
   **в нужной теме** (reply не нужен), бот ставит 👍 при успешной отправке.

Команды:

| Команда | Где | Что делает |
|---|---|---|
| `/add` | личка | добавить MAX-аккаунт (телефон + SMS + 2FA) |
| `/accounts` | личка | список твоих аккаунтов и статус |
| `/remove N` | личка | удалить аккаунт N (сессию и темы) |
| `/bind [N]` | группа | привязать группу к аккаунту |
| `/mute` / `/unmute` | тема | заглушить/вернуть звук этого чата |
| `/muted` | группа | список заглушённых чатов |

Несколько аккаунтов у одного пользователя — просто повтори `/add` и привяжи
каждому свою группу.

## Прокси (если сервер в другой стране, чем телефон)

MAX блокирует вход с «чужого» IP (вплоть до отказа слать SMS). Тогда для аккаунта
нужен **HTTP(S)-прокси** с IP в нужном регионе. Сейчас прокси задаётся при
миграции через `.env` (`MAX_PROXY`) или в реестре; для добавляемых через бота
аккаунтов поддержку прокси можно расширить отдельно — напиши, если нужно.

## Запуск как сервис (systemd)

```ini
# /etc/systemd/system/maxtotelega.service
[Unit]
Description=MaxToTelega bridge
After=network-online.target
Wants=network-online.target

[Service]
WorkingDirectory=/opt/maxtotelega/app
ExecStart=/opt/maxtotelega/app/.venv/bin/python -u bridge.py
Restart=always
RestartSec=10
User=maxbridge
MemoryMax=400M
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/opt/maxtotelega/app/data

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now maxtotelega
sudo journalctl -u maxtotelega -f
```

Вход теперь происходит **через Telegram**, поэтому интерактивный запуск в консоли
больше не нужен — можно сразу запускать сервисом. После перезагрузки все аккаунты
поднимаются из реестра автоматически.

## Переезд со старой single-версии

Если раньше был один аккаунт из `.env` — заполни в `.env` блок миграции
(`TELEGRAM_OWNER_ID`, `MAX_PHONE`, `TELEGRAM_GROUP_ID`, при необходимости
`MAX_SESSION`, `MAPPING_DB`, `MAX_PROXY`). При первом запуске с **пустым реестром**
этот аккаунт перенесётся автоматически, с сохранением сессии и тем — повторно
входить не придётся.

## Структура

| Файл | Назначение |
|---|---|
| `bridge.py` | Менеджер: онбординг аккаунтов, маршрутизация, жизненный цикл |
| `registry.py` | Реестр аккаунтов (кто/телефон/группа/сессия) |
| `storage.py` | Per-account база: темы и маршрутизация ответов |
| `config.py` | Общие настройки из `.env` |

## Что умеет

- Текст, фото, файлы, видео, **стикеры** и **аудио** из MAX → Telegram.
- Несколько фото/видео — **альбомом**; нерисуемые вложения — текстовой пометкой.
- Ответы из тем → обратно в MAX (текст, фото, документы, видео).
- Темы автоматически именуются по чату/собеседнику и переименовываются, когда
  появляется имя; чаты-боты помечаются `🤖 бот`.
- Тихие чаты (`/mute`), авто-ротация логов, чистое завершение, авто-перезапуск
  аккаунтов после рестарта.

## Ограничения

- Отметка «прочитано» отключена (текущий MAX рвёт на неё соединение).
- Контакты, опросы, «кружки» из MAX приходят пометкой, не разворачиваются.
- Анимированные стикеры (Lottie) — статичной картинкой.
- Видео/файлы ограничены лимитами Telegram Bot API (~50 МБ на отправку ботом).
- Прокси для аккаунтов, добавленных через бота, пока задаётся только через
  реестр/миграцию (не из диалога).
